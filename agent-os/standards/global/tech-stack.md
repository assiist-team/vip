## Tech stack

Define your technical stack below. This serves as a reference for all team members and helps maintain consistency across the project.

### Framework & Runtime
- **Application Framework:** Flutter
- **Language/Runtime:** Dart (Flutter), Python
- **Package Manager:** pub (Flutter/Dart), pip (Python)

### Frontend
- **UI Framework:** Flutter (web/desktop)
- **Styling:** Flutter Material/Cupertino widgets, custom themes
- **State Management:** Riverpod

### Database & Storage
- **Database:** Supabase (PostgreSQL)
- **Database Operations:** Cursor MCP for Supabase
- **Storage:** Supabase Storage

### Backend & Serverless
- **Cloud Functions:** Supabase Edge Functions (if needed)

### Testing & Quality
- **Test Framework:** Flutter Test, pytest (Python)
- **Linting/Formatting:** flutter_lints (Dart), Black and Flake8 (Python)

### Deployment & Infrastructure
- **Hosting:** 
  - Web: Cloudflare Pages
  - Mobile: App Store (iOS) and Google Play Store (Android)
  - Desktop: GitHub Releases or custom distribution
- **CDN/DNS:** Cloudflare
- **CI/CD:** GitHub Actions

### Third-Party Services
- **Authentication:** Supabase Auth
- **Database & Backend:** Supabase
- **Email:** Supabase transactional email (SMTP)
- **Monitoring:** Sentry
